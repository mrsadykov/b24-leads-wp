# Заявки в Битрикс24 (WordPress)

Плагин WordPress для отправки заявок с форм сайта в Битрикс24 как **лиды** или **сделки** по входящему вебхуку.

## Требования

- WordPress 5.0+
- PHP 7.4+
- Портал Битрикс24 с доступом к CRM и входящим вебхукам (раздел «Разработчикам»)

## Установка

1. Скачайте или клонируйте плагин в `wp-content/plugins/b24-leads-wp/`.
2. В админке WordPress: **Плагины → Добавить новый → Загрузить плагин** (если у вас zip) или включите плагин **«Заявки в Битрикс24»** из списка.
3. Перейдите в **Настройки → B24 Заявки** и укажите URL входящего вебхука Битрикс24.

## Настройка вебхука в Битрикс24

1. В Битрикс24: **Настройки → Инструменты → Разработчикам** (Developer resources).
2. Создайте **входящий вебхук** (Incoming webhook).
3. Выберите права доступа: **CRM** (как минимум создание лидов/сделок).
4. Скопируйте **URL вебхука** — он выглядит так:  
   `https://ваш-портал.bitrix24.ru/rest/1/xxxxxxxxxxxxxxxx/`  
   Вставляйте URL **без** имени метода в конце (без `crm.lead.add` и т.п.).

## Настройки плагина

- **URL входящего вебхука B24** — вставьте скопированный URL.
- **Куда отправлять** — **Лиды** (`crm.lead.add`) или **Сделки** (`crm.deal.add`). Для сделок можно указать **Этап сделки (STAGE_ID)** (например `NEW`, `PREPARATION` или `C1:NEW` для своей воронки); если оставить пустым, B24 подставит первый этап по умолчанию.
- **Создавать контакт в B24** — если включено, перед созданием лида/сделки создаётся контакт (crm.contact.add) с именем, телефоном и email из заявки; лид или сделка привязывается к этому контакту (CONTACT_ID / CONTACT_IDS).
- **Маппинг полей** — соответствие имён полей формы (name, phone, email, message, title) полям CRM: NAME, PHONE, EMAIL, COMMENTS, TITLE. В блоке **«Дополнительные поля»** можно добавить свои пары (например company → COMPANY_TITLE). Примеры добавления полей в форму CF7 и WPForms — в файле **`docs/EXAMPLES_EXTRA_FIELDS.md`**.

## Страница с формой заявки (пошагово)

Плагин «Заявки в Битрикс24» работает с формами **Contact Form 7**: после отправки данные автоматически уходят в Битрикс24.

### Шаг 1. Установить Contact Form 7

1. В админке WP: **Плагины → Добавить новый**.
2. В поиске введите **Contact Form 7**.
3. Нажмите **Установить** у плагина «Contact Form 7», затем **Активировать**.

### Шаг 2. Создать форму

1. В меню слева: **Контакты → Контактные формы**.
2. Откройте форму по умолчанию («Контактная форма 1») или нажмите **Добавить новую**.
3. В блоке **Форма** оставьте или вставьте разметку с полями (имена полей должны быть как ниже — тогда наш плагин их подхватит):

```
<label> Ваше имя *
    [text* your-name placeholder "Имя"] </label>

<label> Email *
    [email* your-email placeholder "Email"] </label>

<label> Телефон *
    [tel* your-phone placeholder "+7 (999) 123-45-67"] </label>

<label> Сообщение
    [textarea your-message placeholder "Ваше сообщение"] </label>

[submit "Отправить"]
```

4. Сохраните форму.
5. Скопируйте **шорткод** формы (например `[contact-form-7 id="123" title="Контактная форма 1"]`) — он показан над формой.

### Шаг 3. Создать страницу и вставить форму

1. **Страницы → Добавить новую**.
2. Укажите заголовок, например **«Оставить заявку»** или **«Контакты»**.
3. В контент страницы вставьте шорткод формы, например:  
   `[contact-form-7 id="123" title="Контактная форма 1"]`  
   (подставьте свой id и title из Контактов → Контактные формы.)
4. Нажмите **Опубликовать**.

Готово: при отправке формы заявка уйдёт и в письмо (если настроено в CF7), и в Битрикс24 как лид или сделка.

### Важные нюансы по формам

- **Поля в формах** создаются в конструкторе (CF7, Elementor, WPForms и т.д.), а не в настройках плагина. В плагине задаётся только маппинг: какой ключ из формы → в какое поле B24.
- **Пользовательские поля B24 (UF_CRM_*):** если имя поля в форме совпадает с кодом поля в B24 (например `[text UF_CRM_1771489958428 ...]` в CF7), значение уходит в B24 **автоматически**, добавлять пару в «Дополнительные поля» не нужно.
- **Дополнительные поля в маппинге:** сохраняются только заполненные пары; пустые строки после сохранения исчезают. Ключ в форме и «Ключ в форме» в настройках должны совпадать (латиница, без пробелов).
- **Где в B24 посмотреть коды полей:** CRM → Настройки → Настройка полей / Поля лидов (сделок, контактов). Подробные примеры для CF7 и WPForms — в **`docs/EXAMPLES_EXTRA_FIELDS.md`**, пошаговая проверка всех форм — в **`docs/TESTING_FORMS.md`**.

## Подключение форм

### Contact Form 7

Если установлен Contact Form 7, заявки с форм CF7 **отправляются в B24 автоматически** после успешной отправки. Поддерживаются типичные имена полей:

- Имя: `your-name`, `name`, `fio`
- Телефон: `your-phone`, `phone`, `tel`
- Email: `your-email`, `email`
- Сообщение: `your-message`, `message`, `comment`
- Тема: `your-subject`, `subject`

UTM-метки можно передать полями `utm_source`, `utm_medium`, `utm_campaign` и т.д. в форме.

### Elementor Forms (Elementor Pro)

Если установлен **Elementor Pro** с модулем форм, заявки с виджета «Форма» **отправляются в B24 автоматически**. Поля маппятся по типу (email, tel, textarea) и по подписи (имя, телефон, сообщение и т.д.). Данные можно скорректировать фильтром `b24_leads_wp_elementor_data`.

### WPForms

Если установлен **WPForms**, заявки **отправляются в B24 автоматически** после успешной отправки. Поля маппятся по типу: Name → имя, Email → email, Phone → телефон, Textarea → сообщение. Фильтр: `b24_leads_wp_wpforms_data`.

### Gravity Forms

Если установлен **Gravity Forms**, заявки **отправляются в B24 автоматически** после отправки формы. Поля маппятся по типу поля (Name, Email, Phone, Paragraph). Фильтр: `b24_leads_wp_gravity_data`.

### Любая другая форма (Gutenberg, своя тема)

Вызовите в коде после успешной отправки формы:

```php
do_action( 'b24_leads_wp_send_lead', array(
	'name'    => 'Иван',
	'phone'   => '+7 999 123-45-67',
	'email'   => 'ivan@example.com',
	'message' => 'Текст заявки',
	'title'   => 'Заявка с главной',  // необязательно
	'utm_source'   => 'yandex',      // необязательно
	'utm_medium'   => 'cpc',
	'utm_campaign' => 'spring',
) );
```

Или используйте хелпер:

```php
b24_leads_wp_send_lead( array(
	'name'    => $name,
	'phone'   => $phone,
	'email'   => $email,
	'message' => $message,
) );
```

Данные будут отправлены в Битрикс24 согласно настройкам маппинга.

## Журнал отправок (логирование)

На странице **Настройки → B24 Заявки** внизу выводится **Журнал отправок**: последние 50 записей с типом (Успех / Ошибка / Пропуск), временем и сообщением. Для успешных отправок показывается ID созданного лида/сделки в B24. Кнопка **Очистить журнал** удаляет все записи. Хранится до 200 записей. При включённых `WP_DEBUG` и `WP_DEBUG_LOG` те же события пишутся в `wp-content/debug.log`.

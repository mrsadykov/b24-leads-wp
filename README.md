# Заявки в Битрикс24 (WordPress)

Плагин WordPress для отправки заявок с форм сайта в Битрикс24 как **лиды** или **сделки** по входящему вебхуку.

**Распространение:** базовая версия плагина публикуется в каталоге [WordPress.org](https://wordpress.org/plugins/) и доступна бесплатно. Монетизация — расширенная версия (Pro), подписка на обновления и поддержку.

## Требования и установка

- **WordPress** 5.0+, **PHP** 7.4+.
- Портал **Битрикс24** с доступом к CRM и входящим вебхукам (раздел «Разработчикам»).

1. Скачайте или клонируйте плагин в `wp-content/plugins/b24-leads-wp/`.
2. В админке: **Плагины → Добавить новый → Загрузить плагин** или включите плагин **«Заявки в Битрикс24»**.
3. Перейдите в **Настройки → B24 Заявки** и укажите URL входящего вебхука Битрикс24.

## Настройка вебхука и плагина

### Вебхук в Битрикс24

1. В Битрикс24: **Настройки → Инструменты → Разработчикам** (Incoming webhook).
2. Создайте **входящий вебхук**, права: **CRM**.
3. Скопируйте **URL вебхука** (например `https://ваш-портал.bitrix24.ru/rest/1/xxxxxxxxxxxxxxxx/`). Вставляйте URL **без** имени метода в конце (без `crm.lead.add` и т.п.).

### Настройки плагина

- **URL входящего вебхука B24** — вставьте скопированный URL.
- **Куда отправлять** — **Лиды** (`crm.lead.add`) или **Сделки** (`crm.deal.add`). Для сделок — **Этап сделки (STAGE_ID)** (например `NEW`, `PREPARATION`, `C1:NEW`); пусто — первый этап по умолчанию.
- **Создавать контакт в B24** — перед лидом/сделкой создаётся контакт и привязывается к заявке.
- **Маппинг полей** — соответствие имён полей формы полям CRM (NAME, PHONE, EMAIL, COMMENTS, TITLE). В **Дополнительные поля** — свои пары (например company → COMPANY_TITLE). Поля с именами `UF_CRM_*` уходят в B24 автоматически.

## Страница с формой заявки

### Contact Form 7: установка и форма

1. **Плагины → Добавить новый** → установите и активируйте **Contact Form 7**.
2. **Контакты → Контактные формы** → откройте форму или **Добавить новую**.
3. В блоке **Форма** вставьте разметку (шаблон с именем, e-mail, темой, сообщением и пользовательским полем):

```
<label> Ваше имя
    [text* your-name autocomplete:name] </label>

<label> Ваш e-mail
    [email* your-email autocomplete:email] </label>

<label> Тема
    [text* your-subject] </label>

<label> Ваше сообщение (не обязательно)
    [textarea your-message] </label>

<label> Новая строка
    [text UF_CRM_1772535878378] </label>

[submit "Отправить"]
```

Поле «Новая строка» (код в B24: `UF_CRM_1772535878378`) уйдёт в CRM автоматически. Маппинг: your-name → NAME, your-email → EMAIL, your-subject → TITLE, your-message → COMMENTS. Для телефона добавьте `[tel* your-phone]` (your-phone → PHONE).

4. Сохраните форму, скопируйте шорткод (например `[contact-form-7 id="123" title="Контактная форма 1"]`).
5. **Страницы → Добавить** → вставьте шорткод в контент → **Опубликовать**.

### Важные нюансы по формам

- Поля создаются в конструкторе формы (CF7, Elementor, WPForms и т.д.); в плагине задаётся только маппинг.
- **UF_CRM_*:** если имя поля в форме совпадает с кодом поля в B24 (например `[text UF_CRM_1772535878378 ...]`), значение уходит в B24 автоматически.
- Дополнительные поля в маппинге: сохраняются только заполненные пары; ключ в форме и «Ключ в форме» в настройках должны совпадать (поддерживаются кириллица и пробелы).
- Коды полей B24: **CRM → Настройки → Настройка полей** / Поля лидов (сделок).

## Подключение форм

- **Contact Form 7** — заявки отправляются в B24 автоматически. Поддерживаются your-name, your-email, your-phone, your-message, your-subject и др.
- **Elementor Forms (Elementor Pro)** — автоотправка; маппинг по типу поля и подписи. Фильтр: `b24_leads_wp_elementor_data`.
- **WPForms** — автоотправка; маппинг по типу (Name, Email, Phone, Paragraph). Фильтр: `b24_leads_wp_wpforms_data`.
- **Gravity Forms** — автоотправка; маппинг по типу. Фильтр: `b24_leads_wp_gravity_data`.
- **Любая форма / свой код:** вызовите `do_action( 'b24_leads_wp_send_lead', $data );` или `b24_leads_wp_send_lead( $data );`, где `$data` — массив с ключами `name`, `phone`, `email`, `message`, `title` (и при необходимости `utm_source` и т.д.).

## Журнал отправок

На странице **Настройки → B24 Заявки** внизу — **Журнал отправок**: последние 50 записей (тип Успех/Ошибка/Пропуск), время, сообщение, ID лида/сделки. Кнопка **Очистить журнал**. Хранится до 200 записей. При `WP_DEBUG` и `WP_DEBUG_LOG` события пишутся в `wp-content/debug.log`. Блок **Последний ответ Битрикс24** — диагностика (код ответа, тело).

# Пошаговая проверка функционала бесплатного плагина

Руководство для полной проверки: настройки, отправка в B24, маппинг, журнал, интеграции с формами.

## Подготовка

- WordPress 5.0+, PHP 7.4+. Портал B24 с входящими вебхуками. Плагин «Заявки в Битрикс24» установлен и активирован.

## Часть 1. Настройки в админке WordPress

### Шаг 1.1. Открытие страницы настроек

**Настройки → B24 Заявки.** Должны быть видны блоки: вебхук, тип сущности, создание контакта, этап сделки, маппинг, дополнительные поля, журнал.

### Шаг 1.2. Настройка вебхука B24

В B24: **Настройки → Инструменты → Разработчикам** → входящий вебхук, права CRM → скопировать URL. Вставить в поле «URL входящего вебхука B24» → **Сохранить изменения**.

### Шаг 1.3. Выбор сущности: лиды или сделки

**Лиды (crm.lead.add)** или **Сделки (crm.deal.add)**. Для сделок: **ID воронки (CATEGORY_ID)** (0 — по умолчанию; 1, 2, 3… — своя воронка), **Этап сделки (STAGE_ID)** (например `NEW`, `PREPARATION`, `C1:NEW`). Этап без префикса — воронка 0; этап `C1:NEW` — воронка с ID 1.

**Как узнать ID воронки и код этапа в B24:** CRM → Сделки → настройки воронки / Воронки продаж. У этапа — системное имя (STAGE_ID). Для не-воронки по умолчанию: префикс `C` + ID воронки + `:` + код (например `C1:NEW`).

### Шаг 1.4. Создание контакта в B24

Включите галочку «Создавать контакт в B24» → сохраните. После отправки заявки в B24 появятся контакт и привязка к лиду/сделке (CONTACT_ID / CONTACT_IDS).

### Шаг 1.5. Маппинг полей формы → Битрикс24 и UF_CRM_*

В блоке «Маппинг полей формы → Битрикс24»: таблица стандартных полей (имя, телефон, email, сообщение, заголовок) и **Дополнительные поля** (кнопка «Добавить поле»). Ключ в форме → поле в B24. В B24 коды полей: **CRM → Настройки → Настройка полей**. Поля с именем вида `UF_CRM_*` в форме уходят в B24 автоматически, в маппинг добавлять не нужно. Кнопка **Сбросить маппинг** — сброс стандартных полей по умолчанию и очистка дополнительных.

## Часть 2. Отправка заявок с форм

### Пример полей для маппинга (дополнительные поля B24)

| Название поля в B24 | Код поля в B24        | В настройках плагина |
|---------------------|------------------------|----------------------|
| Новая строка        | UF_CRM_1772535878378   | Ключ: **Новая строка**, поле: **UF_CRM_1772535878378** |

В форме создайте поле с именем `Новая строка` или `UF_CRM_1772535878378`; при имени = коду B24 маппинг не нужен.

### Шаблон формы (имя, e-mail, тема, сообщение)

- **Contact Form 7:** см. блок кода выше в разделе «Страница с формой заявки».
- **WPForms:** поля Name (your-name), Email (your-email), Single Line «Тема» (your-subject), Paragraph «Сообщение» (your-message); при необходимости Field Name и Дополнительные поля в настройках плагина.
- **Elementor:** виджет Form — поля Текст (your-name), Email, Текст (your-subject), Textarea (your-message).
- **Gravity Forms:** Name, Email, Single Line «Тема», Paragraph «Сообщение»; при отличии имён — Дополнительные поля в плагине.

### Шаг 2.1–2.4. Contact Form 7, WPForms, Elementor, Gravity Forms

Установите нужный конструктор, создайте форму с полями (имя, email, телефон/тема, сообщение), вставьте на страницу, отправьте тестовую заявку. Проверка: **Настройки → B24 Заявки → Журнал отправок** (запись Успех/Ошибка) и **B24 → CRM → Лиды/Сделки** (новая карточка).

### Шаг 2.5. Отправка через хук (любая форма)

`do_action( 'b24_leads_wp_send_lead', $data );` или `b24_leads_wp_send_lead( $data );`. Ключи `$data`: `name`, `phone`, `email`, `message`, `title`, при необходимости `utm_source` и др. Пример разовой отправки (тест): по GET-параметру вызвать отправку с тестовыми данными и проверить журнал и B24. Пример своей формы: в обработчике POST после проверки nonce и санитизации собрать массив и вызвать `b24_leads_wp_send_lead( $data );`.

## Часть 3. Журнал отправок и диагностика

**Журнал:** до 200 записей, на странице — последние 50. **Очистить журнал** — кнопка в блоке. **Последний ответ Битрикс24** — время, метод, HTTP-код, тело ответа (при ошибке — детали от B24).

## Часть 4. Граничные случаи

Пустой/неверный вебхук → Пропуск или Ошибка в журнале. Пустые обязательные данные → при отказе B24 запись Ошибка с деталями.

## Краткий чеклист проверки

| № | Что проверить |
|---|----------------|
| 1 | Страница настроек, сохранение вебхука, типа сущности, этапа |
| 2 | Создание контакта в B24, маппинг и дополнительные поля, UF_CRM_* |
| 3 | CF7, WPForms, Elementor, Gravity → B24, хук b24_leads_wp_send_lead |
| 4 | Журнал: записи, очистка, диагностика «Последний ответ B24» |
| 5 | Ошибки при неверном вебхуке / пустых данных |

# Примеры: дополнительные поля и UF_CRM_*

Чтобы в B24 попадали свои поля: 1) добавить поле в форму (CF7/WPForms и т.д.); 2) либо имя поля = код B24 (для `UF_CRM_*` — маппинг не нужен), либо в настройках плагина добавить пару **Ключ в форме** → **Поле в B24**.

**UF_CRM_*:** если имя поля в форме совпадает с кодом поля в B24 (например `UF_CRM_1772535878378`), плагин отправляет его в B24 автоматически.

## Contact Form 7

**Базовый шаблон** — см. раздел «Страница с формой заявки» выше (your-name, your-email, your-subject, your-message, UF_CRM_1772535878378).

**Добавить своё поле (например «Компания»):** в разметке формы добавить `[text company placeholder "Название компании"]`. В **Настройки → B24 Заявки → Дополнительные поля**: ключ `company`, поле B24 `COMPANY_TITLE` → Сохранить.

## WPForms

Добавить поле (Single Line), в настройках поля в блоке **Расширенные** указать **Имя поля** (Field Name) `company`. В плагине в Дополнительные поля: `company` → `COMPANY_TITLE`.

## Кратко

| Конструктор   | Где ключ поля              | В маппинге плагина   |
|---------------|----------------------------|------------------------|
| Contact Form 7 | В разметке: `[text company ...]` | Ключ: company, B24: COMPANY_TITLE |
| WPForms       | Настройки поля «Имя поля»  | Ключ: company, B24: COMPANY_TITLE |

Поля с именами `UF_CRM_*` из формы добавляются к запросу в B24 автоматически. Коды полей в B24: **CRM → Настройки → Настройка полей** / Поля лидов (сделок, контактов).

# Пошаговая проверка интеграции форм

Общая подготовка: WordPress, плагин «Заявки в Битрикс24» включён, в **Настройки → B24 Заявки** указан URL вебхука и тип (Лиды/Сделки). После каждой отправки проверять журнал плагина и CRM в B24.

- **Contact Form 7:** Контакты → Контактные формы → форма с your-name, your-email, your-phone, your-message → страница со шорткодом → отправка → журнал + B24.
- **Elementor Forms:** Elementor + Elementor Pro, виджет Form, поля Имя/Email/Телефон/Сообщение → публикация → отправка → журнал + B24.
- **WPForms:** WPForms → Добавить новую → шаблон или Blank, поля Name, Email, Phone, Paragraph → страница с формой → отправка → журнал + B24.
- **Gravity Forms:** Формы → Новая форма → поля Name, Email, Phone, Paragraph → шорткод на страницу → отправка → журнал + B24.

**Если заявка не попала в B24:** смотреть журнал (Ошибка/Пропуск), блок «Последний ответ Битрикс24» (тело ответа), проверить URL вебхука и права, наличие полей по маппингу в форме.

# Локальный тест

**Local (Local by Flywheel):** скачать с [localwp.com](https://localwp.com/), создать сайт, Start site → WP Admin. Скопировать папку `b24-leads-wp` в `app/public/wp-content/plugins/` → Плагины → активировать. Опционально установить Contact Form 7. Настройки → B24 Заявки → URL вебхука. Отправить форму или вызвать `b24_leads_wp_send_lead( array(...) );` — в B24 должен появиться лид/сделка.

**Docker:** `docker-compose.yml` с сервисами `wp` (wordpress:latest) и `db` (mysql:5.7), volume плагина в `wp-content/plugins/b24-leads-wp`. Запуск: `docker compose up -d`, установка WP в браузере, активация плагина и настройка вебхука.

**Ручная установка:** скачать WordPress, база MySQL, распаковать в каталог веб-сервера, установить WP, скопировать плагин в `wp-content/plugins/b24-leads-wp/`, активировать и настроить вебхук.

Для проверки B24 нужен портал с входящим вебхуком (Настройки → Разработчикам → права CRM). Локальный WP не мешает: запрос идёт на серверы B24.

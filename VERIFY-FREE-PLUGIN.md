# Пошаговая проверка функционала бесплатного плагина «Заявки в Битрикс24»

Руководство для полной проверки всех возможностей бесплатной версии плагина: настройки, отправка в B24, маппинг, журнал, интеграции с формами.

---

## Подготовка

- WordPress 5.0+, PHP 7.4+.
- Портал Битрикс24 с доступом к CRM и входящим вебхукам (Настройки → Инструменты → Разработчикам).
- Установлен и активирован плагин «Заявки в Битрикс24» (папка `b24-leads-wp` в `wp-content/plugins/`).

---

## Часть 1. Настройки в админке WordPress

### Шаг 1.1. Открытие страницы настроек

1. Войдите в админку WordPress.
2. В меню слева: **Настройки** → **B24 Заявки**.
3. Должна открыться страница с полями: URL вебхука, куда отправлять, маппинг полей и т.д.

**Ожидание:** страница загружается без ошибок, видны все блоки (вебхук, тип сущности, создание контакта, этап сделки, маппинг, дополнительные поля, журнал).

---

### Шаг 1.2. Настройка вебхука B24

1. В Битрикс24: **Настройки** → **Инструменты** → **Разработчикам** (или **Настройки** → **Другое** → **Входящий webhook**).
2. Создайте **входящий вебхук**, права: **CRM**.
3. Скопируйте **URL** (например `https://ваш-портал.bitrix24.ru/rest/1/xxxxxxxx/`). URL должен быть **без** суффикса метода (без `/crm.lead.add`, без `.json`).
4. На странице **Настройки → B24 Заявки** вставьте URL в поле **«URL входящего вебхука B24»**.
5. Нажмите **Сохранить изменения**.

**Ожидание:** настройки сохраняются, сообщение об успешном сохранении. При неверном URL плагин всё равно сохранит значение; ошибка проявится при отправке формы (см. ниже).

---

### Шаг 1.3. Выбор сущности: лиды или сделки

1. В блоке **«Куда отправлять»** выберите **Лиды (crm.lead.add)**.
2. Сохраните. Отправьте тестовую заявку с сайта (см. часть 2) — в B24 должен появиться **лид**.
3. Вернитесь в настройки, выберите **Сделки (crm.deal.add)**.
4. Чтобы заявки попадали **в нужную воронку**, укажите **ID воронки (CATEGORY_ID)**:
   - **0** — воронка по умолчанию (**общая воронка**). Сделки без указания воронки попадают сюда.
   - **1, 2, 3…** — ID вашей воронки. Как узнать ID — см. подраздел ниже.
5. Укажите **Этап сделки (STAGE_ID)** (например `NEW`, `PREPARATION` или для своей воронки `C1:NEW`, `C2:PREPARATION`). Пусто — B24 подставит первый этап.
   - **Важно:** если этап указан **без префикса воронки** (например только `PREPARATION`, `NEW`), сделки попадают в **воронку 0 (общая воронка)**. Если этап вида `C1:NEW` — в воронку с ID 1; плагин может подставить ID воронки по префиксу этапа, либо укажите **ID воронки** явно.
6. Сохраните. Отправьте тестовую заявку — в B24 должна появиться **сделка** в выбранной воронке и этапе.

**Ожидание:** заявки создаются в B24 как лиды или как сделки; сделки с этапом без префикса (например `PREPARATION`) — в воронку 0; с этапом `C1:NEW` и т.п. — в соответствующую воронку.

#### Как в Битрикс24 Облако узнать ID воронки и код этапа сделки

- **ID воронки (CATEGORY_ID):**
  1. В Битрикс24: **CRM** → **Сделки**.
  2. Переключите воронку на нужную (выпадающий список или вкладка с названием воронки).
  3. Откройте **настройки воронки**: иконка шестерёнки рядом с названием воронки или **Настройки** → **Настройки CRM** → **Воронки продаж** (или **Направления сделок**).
  4. В списке воронок у каждой есть **ID** (число). Воронка «По умолчанию» обычно имеет ID **0**, остальные — **1**, **2**, **3** и т.д. Если в интерфейсе ID не виден, его можно получить через REST API (метод `crm.category.list`).
- **Код этапа (STAGE_ID):**
  1. В том же разделе **Воронки продаж** откройте нужную воронку (редактирование).
  2. Перейдите к списку **этапов** (стадий) воронки.
  3. У каждого этапа есть **системное имя** или **код** — это и есть STAGE_ID (например `NEW`, `PREPARATION`, `CLOSED_WON`).
  4. Для **не воронки по умолчанию** код этапа указывается с префиксом воронки: **C** + ID воронки + **:** + код этапа, например для воронки с ID 1 первый этап — `C1:NEW`, второй — `C1:PREPARATION`.

### Шаг 1.4. Создание контакта в B24

1. Включите галочку **«Создавать контакт в B24»** (перед лидом/сделкой создаётся контакт и привязывается к заявке).
2. Сохраните.
3. Отправьте заявку с формы, в которой есть имя, телефон или email.
4. В Битрикс24 проверьте: в разделе **Контакты** должен появиться новый контакт; у созданного лида/сделки должен быть привязан этот контакт (CONTACT_ID / CONTACT_IDS).

**Ожидание:** контакт создаётся, лид/сделка связаны с ним.

### Шаг 1.5. Маппинг полей формы → Битрикс24 и пользовательские поля (UF_CRM_*)

В блоке **«Маппинг полей формы → Битрикс24»** на странице **Настройки → B24 Заявки** задаётся соответствие: **имя поля в форме** (ключ) → **код поля в B24**. Одна таблица: сначала стандартные поля (имя, телефон, email, сообщение, заголовок), затем **уже добавленные дополнительные поля** — каждая пара отображается строкой с кнопкой **«Удалить»**; новые пары добавляются кнопкой **«Добавить поле»**.

#### В Битрикс24: какие поля есть и где взять коды

1. Откройте портал B24 → **CRM** → **Настройки** (или **Настройки** → **Настройки CRM**).
2. Раздел **«Настройка полей»** (или **«Поля лидов»** / **«Поля сделок»**). Там перечислены стандартные поля (NAME, PHONE, EMAIL, COMMENTS, TITLE и т.д.) и пользовательские поля с кодами вида **UF_CRM_1234567890**.
3. Запомните или скопируйте код поля B24, которое хотите заполнять с сайта (например `COMPANY_TITLE` для компании, `UF_CRM_1771489958428` для своего поля).

#### В админке WordPress: стандартные пары и дополнительные поля

1. **Настройки** → **B24 Заявки**, прокрутите до блока **«Маппинг полей формы → Битрикс24»**.
2. В **таблице стандартных полей** указаны пары:
   - **Имя (форма)** → поле B24 (по умолчанию NAME);
   - **Телефон** → PHONE;
   - **Email** → EMAIL;
   - **Сообщение** → COMMENTS;
   - **Заголовок заявки** → TITLE.
   При необходимости измените **правый столбец** (код поля B24). Имена полей в форме (your-name, fio, phone и т.д.) плагин сопоставляет с этими ключами (name, phone, email, message, title) в зависимости от типа формы; если у вас своё поле (например «Компания» с именем `company`), его нужно добавить в Дополнительные поля (см. ниже).
3. В подблоке **«Дополнительные поля (по желанию)»** отображается таблица: **Ключ в форме** | **Поле в Битрикс24**. Все добавленные вами пары видны в этой таблице. Нажмите **«Добавить поле»** — появится новая строка. Пример: ключ в форме `company`, поле в B24 `COMPANY_TITLE`. Заполните обе колонки, нажмите **Сохранить изменения** — пара останется в таблице и будет использоваться при отправке.
4. Нажмите **Сохранить изменения**.

#### В форме на сайте: имена полей

5. В конструкторе формы (CF7, WPForms, Elementor и т.д.) у каждого поля есть **имя (name)**. Для стандартных полей плагин распознаёт типичные имена (в CF7 — your-name, your-email, your-phone, your-message; в других конструкторах — по типу поля или подписи). Для полей из **Дополнительные поля** имя поля в форме должно **точно совпадать** с ключом (например поле «Компания» с именем `company` → данные уйдут в COMPANY_TITLE в B24).

#### Пользовательские поля B24 (UF_CRM_*): без добавления в маппинг

6. Если нужно заполнять **пользовательское поле** B24 с кодом вида `UF_CRM_1234567890`:
   - В B24: **CRM** → **Настройки** → **Настройка полей** (лидов или сделок) → создайте или найдите поле, скопируйте его код (UF_CRM_...).
   - В форме на сайте добавьте поле с **именем (name)**, в точности равным этому коду (например `UF_CRM_1771489958428`). В **Дополнительные поля** в настройках плагина эту пару добавлять **не нужно** — плагин автоматически отправляет все поля с префиксом `UF_CRM_` в B24.
   - Отправьте заявку — значение попадёт в соответствующее пользовательское поле лида/сделки.

**Ожидание:** в блоке «Маппинг полей» отображаются стандартные пары и все добавленные дополнительные поля; данные из формы попадают в нужные поля CRM (стандартные и UF_CRM_*) согласно маппингу и именам полей.

## Часть 2. Отправка заявок с форм

### Шаг 2.1. Contact Form 7

**В админке WordPress:**

1. Установите и активируйте плагин **Contact Form 7** (Плагины → Добавить новый → поиск «Contact Form 7»).
2. В меню слева: **Контакты** → **Контактные формы**.
3. Создайте новую форму или откройте существующую для редактирования.
4. В теле формы должны быть поля с именами, которые плагин B24 распознаёт: например `[text* your-name]`, `[email* your-email]`, `[tel your-phone]`, `[textarea your-message]`. Стандартный маппинг плагина связывает your-name → NAME, your-email → EMAIL, your-phone → PHONE, your-message → COMMENTS. При необходимости добавьте поля (например `[text company]`) и укажите пару company → COMPANY_TITLE в **Настройки → B24 Заявки** в блоке «Дополнительные поля».
5. Сохраните форму. Скопируйте шорткод формы (например `[contact-form-7 id="123" title="Контактная форма 1"]`).
6. Создайте или откройте страницу (Записи → Добавить / Редактировать), вставьте шорткод в контент, опубликуйте страницу.

**Проверка отправки:**

7. Откройте эту страницу на сайте (как посетитель). Заполните все поля формы и нажмите кнопку отправки.
8. **В WordPress:** зайдите в **Настройки** → **B24 Заявки**, прокрутите вниз до блока **«Журнал отправок»**. Должна появиться новая строка: время, тип **Успех** (или Ошибка), сообщение (например «Создан лид в B24 (ID: 123)»).
9. **В Битрикс24:** откройте **CRM** → **Лиды** (или **Сделки**, если в настройках плагина выбраны сделки). Найдите последнюю заявку по времени, откройте её и убедитесь, что имя, телефон, email, сообщение и другие переданные поля заполнены корректно.

**Ожидание:** заявка уходит в B24 сразу после успешной отправки формы CF7; запись в журнале и новая карточка в B24.

### Шаг 2.2. WPForms

**В админке WordPress:**

1. Установите и активируйте **WPForms** (Плагины → Добавить новый).
2. В меню: **WPForms** → **Add New** (или **Добавить новую**). Выберите шаблон (например «Simple Contact Form») или пустую форму.
3. Добавьте поля: **Name** (тип Name), **Email** (Email), **Phone** (Phone), **Paragraph** или **Text** для сообщения (Paragraph/Text). Плагин B24 сопоставляет их по типу поля: name → NAME, email → EMAIL, phone → PHONE, textarea/текст сообщения → COMMENTS. Для своих полей (например «Компания») добавьте поле с понятным именем (name) и при необходимости продублируйте маппинг в **Настройки → B24 Заявки** (Дополнительные поля).
4. Сохраните форму. В настройках формы выберите **«Embed»** (Встроить) или скопируйте шорткод и вставьте его на нужную страницу. Опубликуйте страницу.

**Проверка отправки:**

5. Откройте страницу с формой на сайте, заполните поля и отправьте форму.
6. **В WordPress:** **Настройки** → **B24 Заявки** → блок **«Журнал отправок»**. Должна появиться запись об успешной отправке (или ошибке).
7. **В Битрикс24:** **CRM** → **Лиды** (или **Сделки**) — откройте последнюю заявку и проверьте заполнение полей.

**Ожидание:** заявка отправляется в B24 автоматически после успешной отправки формы WPForms; данные попадают в B24 по маппингу (тип поля и при необходимости дополнительные поля).

### Шаг 2.5. Отправка через хук (любая форма или свой код)

Плагин предоставляет хук **`b24_leads_wp_send_lead`** и хелпер **`b24_leads_wp_send_lead( $data )`**: вы передаёте массив данных (имя, телефон, email, сообщение и т.д.), плагин применяет маппинг из настроек и отправляет заявку в B24. Подходит для собственных форм, обработки после другой отправки или тестового скрипта.

#### Куда вставить код

- **Вариант А:** в файл темы `functions.php` (дочерняя тема предпочтительнее, чтобы обновления темы не затирали правки).
- **Вариант Б:** отдельный плагин или код-сниппет (например через плагин Code Snippets).

#### Формат данных

Ключи массива `$data` (в нижнем регистре), которые плагин понимает по умолчанию и по маппингу: `name`, `phone`, `email`, `message`, `title`. Дополнительные ключи (например `company`, `utm_source`) обрабатываются через маппинг или отправляются в B24, если заданы в настройках или совпадают с полями B24 (в т.ч. UF_CRM_*).

#### Пример 1: разовая отправка из кода (тест или крон)

Вставьте в `functions.php` (или свой плагин) и один раз откройте любую страницу сайта в браузере (или вызовите по крону), чтобы выполнился код:

```php
add_action( 'init', function () {
    // Выполнять только один раз для проверки (удалите или уберите условие после теста)
    if ( ! isset( $_GET['b24_test_lead'] ) || ! current_user_can( 'manage_options' ) ) {
        return;
    }

    $data = array(
        'name'    => 'Иван Тестов',
        'phone'   => '+7 999 123-45-67',
        'email'   => 'ivan@example.com',
        'message' => 'Тестовая заявка из скрипта',
        'title'   => 'Заявка с главной',
    );

    do_action( 'b24_leads_wp_send_lead', $data );
    // Или: b24_leads_wp_send_lead( $data );

    wp_die( 'Отправлено. Проверьте журнал в Настройки → B24 Заявки и раздел Лиды/Сделки в B24.' );
}, 5 );
```

После сохранения откройте в браузере: `https://ваш-сайт.ru/?b24_test_lead=1` (обязательно будучи авторизованным администратором). В журнале **Настройки → B24 Заявки** должна появиться запись, в B24 — новая заявка.

#### Пример 2: отправка после своей формы (обработчик отправки)

Допустим, у вас своя форма в шаблоне темы: поля с именами `user_name`, `user_phone`, `user_email`, `user_message`. При отправке форма отправляет POST-запрос на ту же страницу или на отдельный обработчик. В обработчике после проверки данных (nonce, санитизация) вызовите хук:

```php
// В обработчике отправки формы (например в functions.php по action формы)
add_action( 'wp_loaded', function () {
    if ( $_SERVER['REQUEST_METHOD'] !== 'POST' || empty( $_POST['my_form_submit'] ) ) {
        return;
    }

    if ( ! wp_verify_nonce( isset( $_POST['my_form_nonce'] ) ? $_POST['my_form_nonce'] : '', 'my_form_nonce' ) ) {
        return;
    }

    $name    = isset( $_POST['user_name'] ) ? sanitize_text_field( $_POST['user_name'] ) : '';
    $phone   = isset( $_POST['user_phone'] ) ? sanitize_text_field( $_POST['user_phone'] ) : '';
    $email   = isset( $_POST['user_email'] ) ? sanitize_email( $_POST['user_email'] ) : '';
    $message = isset( $_POST['user_message'] ) ? sanitize_textarea_field( $_POST['user_message'] ) : '';

    if ( $name === '' && $phone === '' && $email === '' ) {
        return; // нет данных для отправки
    }

    $data = array(
        'name'    => $name,
        'phone'   => $phone,
        'email'   => $email,
        'message' => $message,
        'title'   => 'Заявка с формы на странице ' . get_the_title(),
    );

    b24_leads_wp_send_lead( $data );
}, 20 );
```

Замените `my_form_submit`, `my_form_nonce` и имена полей на свои. В HTML формы добавьте скрытое поле nonce: `<?php wp_nonce_field( 'my_form_nonce', 'my_form_nonce' ); ?>` и кнопку с `name="my_form_submit"`. После успешной отправки формы заявка уйдёт в B24; результат смотрите в **Настройки → B24 Заявки** → Журнал и в B24 в Лидах/Сделках.

## Часть 3. Журнал отправок и диагностика

### Шаг 3.1. Просмотр журнала

1. **Настройки** → **B24 Заявки**, прокрутите до блока **«Журнал отправок»**.
2. После каждой отправки формы должна появляться строка: время, тип (Успех / Ошибка / Пропуск), сообщение. Для успешных отправок может отображаться ID созданного лида/сделки.
3. Журнал хранит до 200 записей; на странице отображаются последние 50.

**Ожидание:** все отправки (успешные и с ошибками) видны в таблице.

---

### Шаг 3.2. Очистка журнала

1. В блоке «Журнал отправок» нажмите кнопку **«Очистить журнал»**.
2. Подтвердите отправку формы (если браузер спросит).
3. Таблица журнала должна стать пустой (или отобразить сообщение «Записей пока нет»).

**Ожидание:** журнал очищается.

---

### Шаг 3.3. Последний ответ B24 (диагностика)

1. После хотя бы одной отправки формы внизу страницы настроек появится блок **«Последний ответ Битрикс24 (диагностика)»**.
2. В нём отображаются: время запроса, метод (crm.lead.add / crm.deal.add), HTTP-код ответа. При успехе — код 200; в теле ответа обычно есть `result` с id созданного лида/сделки.
3. При ошибке (неверный вебхук, нет прав, неверные данные) — не 200 и текст ошибки в теле ответа. Разверните «Тело ответа» для деталей.

**Ожидание:** по этому блоку можно понять, успешно ли B24 принял запрос и что вернул сервер.

---

## Часть 4. Граничные случаи и ошибки

### Шаг 4.1. Пустой или неверный вебхук

1. Очистите поле «URL входящего вебхука B24» или введите заведомо неверный URL. Сохраните.
2. Отправьте форму с сайта.
3. В журнале должна появиться запись типа **Пропуск** с сообщением вроде «Вебхук не настроен. Заявка не отправлена.» (или при неверном URL — **Ошибка** с деталями от B24).

**Ожидание:** заявка не уходит в B24, в логе понятная причина.

---

### Шаг 4.2. Пустые обязательные данные

1. Настройте маппинг так, чтобы обязательные для B24 поля (например NAME) не заполнялись (например отключите поле имени в форме или сопоставьте с пустым полем).
2. Отправьте форму с пустым именем (если плагин не требует имя для контакта — возможен успех; для лида B24 может вернуть ошибку).
3. Проверьте журнал: при отказе B24 — запись **Ошибка** с деталями.

**Ожидание:** при невалидных данных B24 возвращает ошибку, плагин логирует её.

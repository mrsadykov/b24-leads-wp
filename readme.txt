Отправка заявок с форм WordPress в Битрикс24 (лиды или сделки) по входящему вебхуку. В каталоге WordPress.org распространяется бесплатно. Расширенные возможности и приоритетная поддержка — по подписке на сайте автора (см. Plugin URI).

== Description ==

Плагин отправляет данные из форм на сайте в CRM Битрикс24 как лиды или сделки. Поддерживаются Contact Form 7, Elementor Forms (Pro), WPForms, Gravity Forms; возможна отправка из любой формы через хук.

**Возможности:**

* Настройка входящего вебхука B24 (URL из настроек портала)
* Выбор сущности: лиды (crm.lead.add) или сделки (crm.deal.add)
* Для сделок — указание этапа воронки (STAGE_ID)
* Опциональное создание контакта в B24 и привязка к лиду/сделке
* Маппинг полей формы на поля CRM; дополнительные поля (в т.ч. пользовательские UF_CRM_*)
* Поля с именами UF_CRM_* автоматически уходят в B24 без настройки маппинга
* Журнал отправок в админке (успех/ошибка/пропуск), диагностика последнего ответа B24

**Поддерживаемые формы:**

* Contact Form 7 — автоматически
* Elementor Forms (Elementor Pro) — автоматически
* WPForms — автоматически
* Gravity Forms — автоматически
* Любая форма — вызов `do_action( 'b24_leads_wp_send_lead', $data );` или `b24_leads_wp_send_lead( $data );`

== Installation ==

1. Установите плагин через «Плагины → Добавить новый → Загрузить» (ZIP) или поместите папку `b24-leads-wp` в `wp-content/plugins/`.
2. Активируйте плагин.
3. В Битрикс24 создайте входящий вебхук: Настройки → Инструменты → Разработчикам → Входящий вебхук, права CRM. Скопируйте URL.
4. В WordPress: Настройки → B24 Заявки — вставьте URL вебхука, выберите «Лиды» или «Сделки», при необходимости настройте маппинг полей.
5. Добавьте форму на страницу (Contact Form 7, WPForms и т.д.) — заявки начнут уходить в B24.

Подробная инструкция и нюансы по формам — в файле README.md в архиве плагина.

== Frequently Asked Questions ==

= Нужен ли платный тариф Битрикс24? =

Входящие вебхуки и REST API доступны на платных тарифах. На бесплатном тарифе раздел «Разработчикам» может быть недоступен — уточняйте в справке B24.

= Поле из формы не попадает в B24 =

Проверьте: 1) для стандартных полей (имя, телефон, email, сообщение) — имена полей в форме должны совпадать с маппингом (your-name, your-phone и т.д. для CF7); 2) для пользовательских полей B24 (UF_CRM_*) — имя поля в форме должно совпадать с кодом поля в B24; 3) для остальных — добавьте пару в «Дополнительные поля» (ключ в форме → поле B24). Пустые пары не сохраняются.

= Где посмотреть коды полей в B24? =

CRM → Настройки → Настройка полей / Поля лидов (сделок, контактов). У пользовательских полей указан код вида UF_CRM_1234567890.

= Есть ли платная версия? =

В каталоге WordPress.org доступна бесплатная версия. Расширенные функции (Pro) и приоритетная поддержка предлагаются по подписке на сайте автора — ссылка в блоке «Plugin URI» на странице плагина.
